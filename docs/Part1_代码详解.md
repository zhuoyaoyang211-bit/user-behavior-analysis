# Part 1 数据工程层 — 概述

## 1. 目标

对原始淘宝用户行为数据（1万用户采样版，12,256,906行）进行清洗、校验、标准化，产出高质量的基础数据集供后续分析使用。

## 2. 输入与产出

| 项目 | 说明 |
|---|---|
| 输入 | 原始CSV（492MB，5列：time, user_id, item_id, item_category, behavior_type） |
| 产出1 | `output/cleaned_data.parquet`（清洗后标准化数据集，12,256,906行×6列） |
| 产出2 | `output/quality_report.txt`（7维度数据质量校验报告） |

## 3. 处理流程

```
分块加载 → 逐块清洗 → 合并 → 全量异常检测 → 质量校验 → 存储为Parquet
```

| 步骤 | 内容 |
|---|---|
| 分块加载 | 100万行/块分批读取，内存占用控制在20MB以内 |
| 数据完整性校验 | 校验item_id与item_category的一对一关系 |
| 字段筛选 | 保留5个必要字段，删除非必要列 |
| 重复检测 | 按(用户,商品,行为,时间)四元组检测，统计重复率49.3%，经确认属合理现象，保留不删除 |
| 业务规则过滤 | 过滤非法行为类型等业务上不可能的记录 |
| 时间标准化 | 统一为datetime格式 |
| 异常用户检测 | IQR法识别异常活跃用户，经爬虫画像验证区分爬虫与重度买家 |
| 质量校验 | 7维度体检报告：清洗摘要/缺失值/取值范围/重复率/分布/重度买家/时间完整性 |

## 5. 代码结构

```
src/
├── common/
│   ├── constants.py        常量定义（行为类型映射等）
│   ├── exceptions.py       自定义异常层级
│   └── logger.py           统一日志工具
├── config.py               配置管理（路径、参数、数据类型映射）
├── data_loader.py          分块加载器
├── data_cleaner.py         清洗器（六步清洗链）
├── data_quality.py         质检器（七维度报告）
└── main.py                 主入口（流水线编排）
```

## 6. 技术要点

- **内存优化**：数据类型压缩（int64→int32/int16/int8），内存占用降低70%
- **分块处理**：100万行/块，避免大数据一次性加载导致内存溢出
- **格式转换**：CSV→Parquet列式存储，读取速度提升5-10倍，文件体积减小约70%
- **异常检测**：IQR统计法则 + 爬虫行为画像双重验证，避免误删高价值用户
